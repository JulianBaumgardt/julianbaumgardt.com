Internet Tester

A multi-round browser test that looks beyond a peak speed number to explain latency, jitter, failed probes, download consistency, responsiveness under load, resolver timing, and stability.

What It Measures

  • Idle latency. Median round-trip time for small uncached HTTPS requests to the nearest Cloudflare edge.
  • Jitter. Median absolute change between consecutive successful idle-latency probes.
  • Loss approximation. The share of HTTPS probes that fail or exceed the timeout. This is not literal IP packet loss.
  • Download and consistency. Aggregate throughput from concurrent byte streams, plus the tenth-percentile sample divided by median throughput.
  • Responsiveness under load. Latency probes continue while downloads are active. The penalty is loaded median minus idle median.
  • Resolver timing. The Cloudflare Worker times an uncached lookup for a random invalid hostname. This describes the edge resolver, not the device's local DNS stack.

Methodology

  • Warm-up is discarded. An initial request establishes the route before measured rounds begin.
  • Multiple rounds. Light runs 2 latency and 1 load round, Standard runs 3 latency and 2 load rounds, and Thorough runs 5 latency and 3 load rounds. Final metrics use medians so one unusually fast or slow round is less influential.
  • Observed spread. Spread is the largest round deviation from the median. Labels describe the actual run and are not laboratory confidence intervals.
  • Concurrent load. The tester uses multiple simultaneous downloads while latency probes run, exposing queueing delay that a single speed number can hide. It discards the first 512 KiB of each stream from throughput timing so request startup and connection ramp-up do not unfairly reduce fast-connection results.
  • User-controlled data. The page estimates maximum data use before the test and never starts automatically.
  • Abortable work. Stop cancels active fetches, and background-tab use is reported because browsers may throttle hidden pages.
  • Bounded downloads. A stalled download is cancelled after two minutes so the page cannot remain stuck indefinitely.
  • Measurement confidence. The result grades the run from sample duration, sample count, failed probes, observed spread, and whether the tab stayed in the foreground. Very fast transfers can finish too quickly for a high-confidence browser estimate.
  • Actual transfer total. The results show how much download data the completed run transferred, alongside the estimate shown before starting.

Browser And Network Limits

  • It is a browser estimate. The tool may be less precise than native applications or dedicated speed-test protocols that can control connections and timing more directly.
  • No ICMP access. Browsers cannot send traditional ping packets, so every latency sample includes HTTPS and browser overhead.
  • Cloudflare-edge scope. This measures the connection to a nearby Cloudflare location. Routes to a game server, workplace VPN, streaming CDN, or another country may differ.
  • DNS is server-side. Browser privacy, caching, and connection reuse prevent a dependable isolated local DNS measurement.
  • Results are approximate. Wi-Fi interference, VPNs, browser scheduling, power saving, other devices, downloads, and background tabs all affect a run.
  • No universal score. The tool keeps real-time responsiveness and bulk throughput separate because connections can be good at one and poor at the other.

Free-Plan Safeguards

  • The Worker accepts only small probes, resolver checks, and downloads capped at 32 MiB per request.
  • Cloudflare-native edge counters limit each client to 300 total test requests and 18 bandwidth-heavy downloads per minute. These counters are server-side and cannot be bypassed by editing the public page code.
  • Responses disable browser and CDN caching, use identity encoding, and permit only this site's origin.
  • The tester sets no cookies and stores no results in local storage. Cloudflare necessarily receives standard connection metadata, and the client IP is used only as the key for the short-lived edge rate limit.
  • The Worker performs minimal work within the free plan's fixed 10 ms CPU budget, with all statistics calculated in the browser.
  • If the endpoint or daily allowance is unavailable, the page stops with a clear service message instead of inventing a result.
