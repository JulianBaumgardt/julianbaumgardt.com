# W11 Optimiser README

This README belongs to the W11 Optimiser project. It covers simple run instructions, safety boundaries, modes, and generated output notes for the Windows 11 optimiser files.

W11 Optimiser is a conservative Windows 11 performance script intended to work across AMD, NVIDIA, Intel, and mixed-GPU systems.

It does not remove apps, disable Defender, disable firewall, disable Windows Update, disable Microsoft Store, disable VBS/Memory Integrity, change HAGS, disable services, disable startup apps, modify BIOS/UEFI, undervolt, overclock, or run internet code.

## Recommended First Run

From the website, the shortest option is:

```powershell
irm julianbaumgardt.com/w11 | iex
```

That downloads a small bootstrapper, fetches the latest `w11-optimiser.ps1`, and opens a menu. The optimiser itself still asks before applying safe changes.

The easiest option is to double-click:

```text
w11-optimiser-launcher.cmd
```

That opens a simple menu.

You can also run direct CMD shortcuts:

```cmd
w11-optimiser-launcher.cmd audit
w11-optimiser-launcher.cmd preview
w11-optimiser-launcher.cmd x
w11-optimiser-launcher.cmd safe
w11-optimiser-launcher.cmd check
w11-optimiser-launcher.cmd report
w11-optimiser-launcher.cmd undo
```

Menu options:

```text
X  SAFE OPTIMISE (NO TEMP/CACHE CLEANUP)
1  PREVIEW
2  AUDIT
3  POST-CHECK
4  OPEN LAST REPORT
5  SAFE OPTIMISE + TEMP/CACHE CLEANUP
6  UNDO LATEST RUN
0  EXIT
```

Or open PowerShell in the folder containing the script.

Audit only:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\w11-optimiser.ps1 -Mode Audit
```

Preview planned changes without applying anything:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\w11-optimiser.ps1 -Mode Preview
```

Safe optimisation without temp/cache cleanup:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\w11-optimiser.ps1 -Mode SafeOptimize -SkipTempCleanup
```

Safe optimisation with old temp/cache cleanup:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\w11-optimiser.ps1 -Mode SafeOptimize
```

If System Restore is disabled and you accept the reduced recovery safety, you can explicitly skip only the restore point step:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\w11-optimiser.ps1 -Mode SafeOptimize -SkipRestorePoint
```

Post-run verification:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\w11-optimiser.ps1 -Mode PostCheck
```

Undo latest run:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\w11-optimiser.ps1 -Mode UndoLatest
```

Open the latest generated report:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\w11-optimiser.ps1 -Mode OpenLastReport
```

## Notes

- `Preview` creates a local HTML report showing current values and planned safe changes without changing settings.
- `OpenLastReport` opens the newest W11 Optimiser `.html` report from the Reports or Runs folders.
- `SafeOptimize` and `UndoLatest` ask for confirmation when run directly. The launcher asks first, then passes `-Force` to avoid a duplicate prompt.
- `SafeOptimize` and `UndoLatest` will request Administrator elevation if needed. If UAC is cancelled, no changes are made.
- Everything generated by the script is saved under `Desktop\W11 Optimiser`.
- Audit and post-check reports are saved as clean black-and-white `.html` pages under `Desktop\W11 Optimiser\Reports`.
- Safe optimisation runs are saved under `Desktop\W11 Optimiser\Runs\Safe Run <timestamp>`.
- Each safe optimisation run creates `Run Report.html`, a readable browser page with a line-by-line checklist, status, warnings, errors, next steps, and a short riskier-tweaks note.
- Safe optimisation, audit, preview, and post-check open their `.html` reports in your default browser automatically.
- New runs do not create `.txt` reports, logs, transcripts, or manual-note files by default. Add `-VerboseLog` to save a timestamped step log in the run folder.
- Essential local backup files are still created so undo remains possible; nothing is uploaded online.
- A restore point is required by default. Use `-SkipRestorePoint` only when System Restore is unavailable and you understand that registry/state backups remain, but the extra Windows restore-point fallback will not exist.
- Safe optimisation enables TRIM and runs a Windows SSD ReTrim maintenance pass on supported fixed NTFS/ReFS volumes.
- Network power management is scoped to active physical adapters, not virtual VPN/Hyper-V-style adapters. It disables the adapter sleep permission where Windows exposes it, including a registry fallback for adapters/drivers that do not expose the setting through `Set-NetAdapterPowerManagement`.
- This can improve stability or throughput on some USB Wi-Fi adapters, but it is hardware/driver dependent and is not a guaranteed internet speed increase.
- Audit and post-check reports include fixed-volume free-space warnings because low free space can affect updates, caches, and game installs.
- On laptops, use the optimised profile while plugged in and check the OEM performance/cooling mode manually.
- GPU-specific settings should be changed only through official AMD, NVIDIA, or Intel control panels.

## Riskier Tweaks

Pros:

- Some riskier tweaks can help specific hardware, drivers, or games.
- Startup cleanup can make login lighter.
- HAGS or VBS changes can sometimes produce measurable differences.

Cons:

- Results are inconsistent across PCs.
- Security-impacting tweaks reduce protection.
- Driver and registry hacks can cause stutter, crashes, or update problems.
- Service and timer tweaks are often hard to reverse and hard to troubleshoot.

Recommendation: keep riskier tweaks manual and measured. Do not bundle them into the safe all-in-one run.
